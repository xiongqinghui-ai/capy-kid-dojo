# 卡皮成长小屋 v2

入口：

- 孩子首页：`index.html`
- 分数记录页：`record.html`
- 家长设置：`parent.html`

## 这版的调整

- 使用孩子头像作为首页主视觉，头像文件在 `assets/avatar.png`。
- 整体改为卡皮巴拉棕色系列，更圆润、活泼、儿童化。
- 显示和设置分离：孩子首页只显示积分、每日完成情况、活动打卡、愿望差距；家长配置单独放在 `parent.html`。
- 首页第一屏显示当前积分、今日加分、今日完成项、当前愿望和每日活动完成图。
- 分数记录页按家长配置的活动分类展示全部活动，孩子可填写完成内容和自评分。
- 家长页可管理孩子昵称、头像路径、活动、愿望、备份导入导出和恢复演示数据。
- 两个页面共享同一份 `localStorage` 数据，后续接云同步时可继续沿用 `app.js` 的数据层。
- 首页标题可在家长页修改，不固定为某个名字。

## 当前同步边界

当前仍是纯前端静态版本，同一浏览器可自动保存。跨 iPad、iPhone、Mac、Windows 的实时同步需要后续接入账号和云数据库。

建议下一步：

1. 增加家庭账号和孩子档案。
2. 把 `KidDojo.save()` 扩展为写入 Supabase、Firebase 或自建 API。
3. 首页订阅云端变更，家长设置页修改后自动推送到所有设备。
4. 增加家长密码或 PIN，避免孩子进入设置页改分数。

## GitHub 发布建议

这个目录是纯静态网页，适合直接发布到 GitHub Pages：

1. 新建一个 GitHub 仓库，例如 `capy-kid-dojo`。
2. 把 `v2` 目录里的文件提交到仓库根目录，或者提交整个 `kid-points-dojo` 后把 Pages 目录设为 `/v2`。
3. 在 GitHub 仓库 Settings -> Pages 中选择 `main` 分支发布。
4. 发布后入口就是 GitHub Pages 给出的网页地址。
